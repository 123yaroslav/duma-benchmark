\section{Оценка безопасности агентной системы на среду исполнения}

\subsection{Бенчмарк кейсов}

Дополнение/создание бенчмарка кейсов для оценки безопасности агентной системы на среду исполнения.

В качестве основы используется T2-бенчмарк (описание исходного бенчмарка подразумевается здесь).

\subsubsection{Добавленные кейсы}

Добавлены следующие кейсы:
\begin{itemize}
    \item атаки через <<коллегу>>;
    \item ресурсный overload;
    \item отравленный RAG;
    \item некорректная обработка выводов.
\end{itemize}

Далее добавляется описание кейсов агентных систем (структура, компоненты, взаимодействия и т.\,д.).

\subsection{Базовые векторы атак}

Описание базовых векторов атак (например, по SAIF-фреймворку):
\begin{itemize}
    \item векторы атак на взаимодействие между агентами;
    \item атаки на каналы ввода/вывода;
    \item атаки на память, RAG и внешние инструменты;
    \item атаки со стороны пользователя / псевдо-пользователя.
\end{itemize}

\section{Цели тестирования и метрики}

\subsection{Цель тестирования}

Цель тестирования~--- оценить устойчивость различных моделей к атакам на агентные системы в реалистичной среде исполнения.

\subsection{Целевые метрики}

Целевые метрики:
\begin{itemize}
    \item $pass_k$~--- прошли ли $k$-asserts в кейсе;
    \item ASR (attack success rate) между разными моделями.
\end{itemize}

Необходимо явно зафиксировать отличия от метрик T2-бенчмарка в рассматриваемых кейсах (например, по трактовке успешности атаки, по учёту стоимости, по штрафам за ложные срабатывания и~т.\,д.).

\section{Изменяемые параметры и условия экспериментов}

\subsection{Изменяемые параметры}

\begin{itemize}
    \item LLM внутри агентной системы (архитектура, размер, семейство моделей).
    \item LLM-юзера (параметры генерации, особенно температура).
\end{itemize}

\subsection{Условия экспериментов}

Условия экспериментов:
\begin{itemize}
    \item объём данных (число кейсов и повторов);
    \item число запусков для каждого сочетания параметров;
    \item параметры, необходимые для того, чтобы убедиться, что результаты замеров обладают статистической значимостью (например, доверительные интервалы, мощность теста).
\end{itemize}

Необходимо дописать условия / оговорку про атакующую модель (характеристики, способности, ограничения).

\subsection{Guardrails}

После получения первых замеров желательно подключить какой-либо guardrail:
\begin{itemize}
    \item GPTOSS;
    \item Llama Guard;
    \item Promptfoo или аналогичный инструмент.
\end{itemize}

\section{Ключевые вопросы тестирования}

Вывод / вопросы тестирования:
\begin{itemize}
    \item Как разные модели противостоят атакам на агентные системы? Какие лучше или хуже?
    \item Есть ли корреляция между размером/семейством модели и устойчивостью к атакам?
    \item Насколько получившиеся кейсы репрезентативны для анализа прикладных ИИ-систем на безопасность?
    \item Желательно дополнить обзором того, как делают другие (например, отличие от Agent Dojo-бенчмарка).
\end{itemize}

Идея для гипотезы:
\begin{itemize}
    \item Проверить на статистическую значимость гипотезу о том, что в зависимости от взаимодействия с пользователем (температура LLM-юзера) меняется секьюрность модели.
\end{itemize}
